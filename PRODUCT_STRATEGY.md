# فيد نيوز — خطة تطوير المنتج

## الوضع الحالي

التطبيق فيه 62 ملف Dart و 7 features رئيسية (auth, content, media, podcast, shell, splash, user). الباكاند PHP متقدم جداً ويحتوي على ميزات AI قوية لم تُستغل بعد في التطبيق.

### ميزات الباكاند الموجودة وغير مستغلة بالتطبيق

| الميزة | الباكاند | التطبيق |
|--------|---------|---------|
| تلخيص AI للأخبار | `cron_ai.php` — ملخصات + نقاط رئيسية + كلمات مفتاحية | غير ظاهرة |
| اسأل الأخبار (AI Q&A) | `ask.php` — سؤال وجواب من الأرشيف | شاشة موجودة لكن أساسية |
| القصص المتطورة AI | `cron_evolving_ai.php` — كيانات + اقتباسات | شاشة أساسية |
| خريطة الأخبار | `cron_news_map.php` + `flutter_map` | شاشة أساسية |
| النشرة الإخبارية | `cron_newsletter.php` | غير موجودة بالتطبيق |
| ملخص تلغرام | `cron_tg_summary.php` | غير موجودة |
| البودكاست + TTS | `cron_podcast.php` + `tts.php` | شاشة أساسية |

## المرحلة 1: تجربة مستخدم احترافية (2-3 أسابيع)

**الهدف:** التطبيق يبدو ويتصرف كمنتج جاهز للإطلاق.

### 1.1 إعادة تصميم الشاشة الرئيسية

الشاشة الرئيسية الحالية عبارة عن قوائم متتالية. المطلوب:

- **شريط أخبار عاجل محسّن:** تحريك أفقي سلس مع عداد (1/5) وألوان تدرجية
- **Hero card تفاعلي:** سحب أفقي لعرض أكثر من خبر رئيسي (3-5 أخبار)
- **بطاقات الأقسام:** تصميم Grid 2×2 بدل الشريط الأفقي — أوضح وأجمل
- **قسم "لك":** أخبار مخصصة حسب اهتمامات المستخدم ومصادره المتابَعة

### 1.2 تصميم بطاقة الخبر

- صورة مصغرة يمين + عنوان + مصدر + وقت بالعربي ("منذ 3 ساعات")
- شريط سفلي: أيقونة المصدر + اسمه + زر حفظ + زر مشاركة
- تأثير سحب: سحب يمين = حفظ، سحب يسار = مشاركة

### 1.3 Dark Mode حقيقي

- ليس مجرد ألوان معكوسة — تدرجات مدروسة للقراءة الليلية
- التبديل التلقائي حسب وقت النظام

### 1.4 تحسين الأداء

- تحميل الصور بـ placeholder ثم fade-in (موجود جزئياً مع cached_network_image)
- Infinite scroll بدل تحميل كل الأخبار دفعة واحدة
- Skeleton loading (موجود لكن يحتاج تحسين)

## المرحلة 2: ميزات AI — القيمة المضافة الحقيقية (3-4 أسابيع)

**الهدف:** كل خبر يصبح "ذكي" — هنا الميزة التنافسية.

### 2.1 الخبر الذكي — Smart Article View

عند فتح أي خبر، يظهر:

- **ملخص AI** (3 أسطر) في أعلى المقال — من حقل `ai_summary` الموجود
- **نقاط رئيسية** (bullets) — من `ai_key_points`
- **سياق القصة:** ربط تلقائي بالقصة المتطورة إن وُجدت
- **مقالات ذات صلة** حسب `ai_keywords`
- **زر "اسأل عن هذا الخبر"** — يفتح AI Q&A مع سياق الخبر محمّل مسبقاً

### 2.2 اسأل الأخبار — AI Chat محسّن

الباكاند جاهز (`ask.php`). التطبيق يحتاج:

- واجهة محادثة حقيقية (chat bubbles) مع streaming response
- أسئلة مقترحة ديناميكية حسب أخبار اليوم
- كل جواب يستشهد بمقالات من الأرشيف (مع روابط قابلة للنقر)
- حفظ تاريخ المحادثات

**هاي الميزة وحدها كافية تخلّي صحفي يثبّت التطبيق** — بدل ما يبحث يدوياً، يسأل: "شو صار بموضوع المعتقلين هالأسبوع؟"

### 2.3 القصص المتطورة — Evolving Stories

الباكاند يستخرج كيانات واقتباسات بالذكاء الاصطناعي. التطبيق يعرضها كـ:

- **Timeline تفاعلي:** خط زمني بصري لتطور القصة
- **جدار الاقتباسات:** أبرز تصريحات المسؤولين مرتبة زمنياً
- **لوحة الأرقام:** "Story by Numbers" — إحصائيات مستخرجة بالـ AI
- **إشعار:** "القصة تطورت" عند إضافة مقال جديد لقصة يتابعها المستخدم

### 2.4 بريفينغ الصباح — Morning Briefing

- إشعار يومي الساعة 7 صباحاً: "صباح الخير — ملخص أخبارك جاهز"
- شاشة ملخص: 5-7 أخبار مع ملخص AI لكل واحد
- خيار "اقرأ لي" — يستخدم TTS الموجود بالباكاند
- قابل للتخصيص: المستخدم يختار أقسام ومصادر البريفينغ

## المرحلة 3: ميزات للصحفيين والنخب (2-3 أسابيع)

**الهدف:** أدوات متخصصة تجعل التطبيق لا غنى عنه للمحترفين.

### 3.1 رادار المصادر — Source Radar

- متابعة مصدر محدد مع إشعار فوري لكل خبر جديد
- مقارنة تغطية: "كيف غطت المصادر المختلفة نفس الحدث؟"
- إحصائيات المصدر: عدد الأخبار يومياً، أوقات الذروة، الأقسام الأنشط

### 3.2 ملفات المواضيع — Topic Dossiers

- المستخدم ينشئ "ملف" لموضوع معين (مثلاً: "الأسرى"، "الانتخابات")
- التطبيق يجمع تلقائياً كل خبر متعلق بالموضوع
- ملخص AI أسبوعي للملف
- تصدير PDF للتقارير

### 3.3 تنبيهات ذكية — Smart Alerts

- بدل إشعار لكل خبر: تنبيه فقط عند حدث مهم فعلاً
- AI يحدد درجة أهمية الخبر (عاجل / مهم / عادي)
- المستخدم يحدد عتبة التنبيه

## المرحلة 4: النمو والانتشار (مستمر)

### 4.1 المشاركة الذكية

- مشاركة خبر كـ "بطاقة" بصرية (صورة مُنسّقة فيها العنوان + المصدر + الشعار)
- رابط عميق: المشاركة تفتح التطبيق مباشرة على الخبر
- ويدجت iOS/Android: أهم 3 أخبار على الشاشة الرئيسية

### 4.2 المجتمع

- تعليقات على الأخبار (moderated)
- "ساهم بتصحيح" — المستخدم يبلّغ عن خطأ بالخبر
- تصويت: "هل هذا الملخص دقيق؟" — يحسّن جودة الـ AI

### 4.3 تحقيق الدخل

- **الطبقة المجانية:** كل الأخبار + ملخصات AI + 5 أسئلة AI يومياً
- **الاشتراك Pro:** أسئلة AI غير محدودة + بريفينغ صباحي + ملفات مواضيع + تنبيهات ذكية + بدون إعلانات
- **اشتراك مؤسسي:** API access + تقارير مخصصة + عدة مستخدمين

## ملاحظة تقنية

أهم شي نحتاج نعمله قبل البدء: **عمل API endpoints في الباكاند** لكل ميزة AI. حالياً الـ AI summaries موجودة بقاعدة البيانات بس ما في endpoint يرجعها مع المقال. المطلوب:
1. إضافة `ai_summary` و `ai_key_points` لـ `api_format_article()` في `/api/v1/helpers.php`
2. إنشاء endpoint جديد: `POST /api/v1/ask` للـ AI Q&A
3. إنشاء endpoint: `GET /api/v1/briefing` للبريفينغ الصباحي
